# 变更日志 (Changelog)

本项目的所有重大变更都记录在此文件中。

本项目遵循 [语义化版本控制 (SemVer)](https://semver.org/spec/v2.0.0.html) 规范。

## [Unreleased]

### Added
- 新增 Swift Package 构建清单与 16 项零依赖核心逻辑测试，覆盖页码范围、端点安全分类和 OpenAI 流式分包，测试可在仅安装 Command Line Tools 的离线环境运行。
- 新增 API 密钥保存与清除控件。密钥仅在用户确认后写入 macOS 钥匙串，避免编辑过程中逐字符执行同步 I/O。
- 为 9 个 SwiftUI 视图文件补充 `#Preview`，并在缺少 Xcode 预览插件的命令行环境中自动隔离预览宏。

### Changed
- 将 PDF 状态管理、后台 PDFKit/Vision 工作器和 4 个纯逻辑模块拆分。主引擎由 817 行降至约 324 行，后台对象改由 Actor 串行访问。
- 使用原生 `ContentUnavailableView`、`Picker`、`ProgressView` 和 `GroupBox` 替换 2 个自绘组件，删除 129 行自定义界面代码。
- 将真实最低系统版本统一为 macOS 14。构建产物的 `minos` 从错误的 26.0 修正为 14.0，并增加发布签名与二进制版本校验。
- 水印候选词改为默认不勾选，避免重复页眉或正文被未经确认地删除。

### Fixed
- 修复 AI 取消令牌未被回调校验的问题。旧任务现在同时校验任务令牌与单页请求编号，不再污染后续任务结果。
- 修复无效页码静默回退为全部页面及零页 PDF 可能创建非法闭区间的问题。错误输入现在直接中止并显示具体原因。
- 修复流式响应末行没有换行符时丢失正文、模型列表忽略 HTTP 状态及错误响应可能持续占用内存的问题。
- 移除无上下文的 OCR 全局替换表，避免将正常的“提取码”等正文词语错误改写。
- 移除提取过程的整篇文本重复缓冲，正文缓存由约 2 份降为 1 份；导出时再按页组装。
- 将已验证的 `AppIcon.icns` 纳入构建资源，修复干净工作区中系统 `iconutil` 拒绝旧式 iconset 导致首次打包失败的问题。

### Security
- 修复 `10.example.com`、`192.168.example.com` 等域名被误判为内网地址的问题，并补齐 IPv6 回环与私有地址处理。
- 外部 AI 地址改为逐地址临时授权，公网 HTTP 禁止发送 API 密钥，模型与正文请求均禁止 HTTP 重定向。
- AI 流式未分行缓冲限制为 1 MiB，HTTP 错误正文限制为 64 KiB，阻止异常端点造成无界内存增长。

## [0.3.0] - 2026-07-03

### Added
- 新增三类 PDF 场景模型：电子文本 + 电子水印、扫描正文 + 电子水印、全扫描件。用户选择从底层模式判断收敛为 3 个明确场景，降低配置决策成本。
- 新增 PDF 主画布与结果检查器布局。原文与 AI 结果合并到右侧检查器，主工作区从四栏缩减为三段式结构。
- 新增统一 AI 提示词构建器。默认提示词、修改留痕指令和水印残留词注入从 3 处重复实现收敛为 1 个入口。
- 新增 OCR 与导出过程的可见错误提示。Vision OCR 报错、页面渲染失败和导出失败会显示在侧边栏错误区，减少静默失败。

### Changed
- 调整扫描正文 + 电子水印的默认处理管线：默认保留原始扫描图像执行 OCR，再过滤水印残留；遮罩改为高级兜底选项，降低正文被遮罩误伤的概率。
- 优化启动页与面板视觉。移除装饰性玻璃图标底座、固定尺寸启动卡和卡片悬浮缩放，界面更接近 macOS 原生工具风格。
- 简化 AI 引擎职责。移除已废弃的后台自动落盘字段和分片写盘空实现，AI 引擎只负责按物理页输出净化文本。
- 调整状态文案。移除界面状态中的符号化提示，改为明确的“错误、完成、已取消”等文本。
- 调整构建脚本图标策略。优先复用已有有效应用图标，避免本地重建时重复触发 `iconutil` 生成失败警告。

### Removed
- 移除无效的 Toolbar 侧边栏折叠按钮。当前布局使用手写 `HSplitView`，旧按钮无法实际折叠左侧面板。
- 移除旧四栏界面组件源码：`RawTextColumn`、`AITextColumn`、`DropZoneView`、`AccessoryBarView`、`LiquidGlassContainer`。源码文件减少 5 个，避免后续维护误改旧入口。

### Fixed
- 修复扫描正文 + 电子水印场景默认开启遮罩导致 OCR 误伤正文的问题。
- 修复高级遮罩开关被场景同步逻辑反复重置的问题。用户手动设置后会保持该偏好，不再被界面刷新覆盖。
- 修复 AI 流式接口将 HTTP 错误或空流误判为空白成功结果的问题。
- 修复导出失败只写入控制台、用户界面无反馈的问题。

## [0.2.0] - 2026-06-18

### Added
- 新增 Toolbar 页码控制，支持上一页、下一页和页码输入回车跳转。

### Changed
- 将 PDF 预览调整为单页横向模式，适配触控板左右翻页。
- 将 PDF 与 AI 处理引擎标记为 `@MainActor`，减少手写主线程派发。
- 将 OCR 调用改为 `withCheckedContinuation`，移除同步等待线程阻塞。
- 在 `PDFPreviewView` 更新周期中重新绑定 Coordinator，修复页码状态同步不稳定的问题。

### Fixed
- 清理 SwiftUI 废弃修饰符，减少编译警告。

## [0.1.0] - 2026-06-17

### Added
- 新增启动导入页，支持点击选择 PDF 与拖放导入。
- 新增四栏联动工作区，支持 PDF 预览、原文结果、AI 结果按物理页同步。
- 新增 AI 净化高级选项，支持修改留痕与水印干扰词传入。
- 新增水印说明与防误伤提示。
- 新增基于 `NSSavePanel` 的原文与 AI 结果导出。

### Changed
- 精简默认 AI 系统提示词，将修改留痕拆分为可选指令。
- 取消后台自动存盘，文本结果改为内存缓存，导出由用户手动触发。
- 将去水印与正文通道配置改为卡片式选择。
- 移除 Toolbar 中的 Finder 定位按钮，降低文件位置与导出状态之间的误解。

### Fixed
- 禁用 Command+N 新建窗口，规避多窗口并发操作同一份处理状态。
- 将 PDF 实例加载移至后台，降低大文件导入时的主线程卡顿风险。
- 将结果显示改为只读文本视图，并对流式输出做增量追加，降低长文本刷新开销。
