# 变更日志 (Changelog)

本项目的所有重大变更都将记录在此文件中。

本项目遵循 [语义化版本控制 (SemVer)](https://semver.org/spec/v2.0.0.html) 规范。

## [1.5.0] - 2026-06-11

### Added
- **智能长文本分片推理 (Context OOM & Truncation 修复)**：实现智能多级退避分片算法，在 1500~2000 字窗口内优先按换行与常见标点符号断句断段，预留硬物理切片保命红线。彻底免除了超长 PDF 一次性发送给本地 AI 时造成的显卡爆显存崩溃或上下文超载被强行截断的重大隐患。
- **串行队列推理与分片追加落盘**：底层 AI 采用串行 FIFO 队列流式跑分片，恒定极低推理负载，低配置显卡亦能流畅应对大作。每个分片推理完立即以追加模式落盘 `_AI净化.txt` 并提供被删重建保护，大幅提升容错性。
- **无遮挡打字机流式回显 UI**：重构了 AI tab 界面展现逻辑，仅在首字吐出前的“热身期”显示大转菊花；一旦收到首字，立刻撤走菊花展示文本编辑器，让文字在用户眼前实时流畅流下。
- **流光分段进度横幅**：在 AI 文本框上方新增紫色常驻横幅，动态提示 `“⚡️ AI 正在接力流式净化中：第 X / Y 段...”` 及 `[取消优化]` 按钮，赋予用户清晰的处理盼头与交互主导权。
- **OCR 大图渲染内存强制释放 (提取 OOM 修复)**：在提取循环体的页面处理外围增加 `autoreleasepool` 物理强放块，确保每页产生的高清 NSImage 和 NSBitmapImageRep 在用完后即刻被销毁，使超长 PDF 提取内存保持恒定平稳。
- **多线程写盘防交叉污染安全防护 (残留时序 Bug 修复)**：设计 `currentExtractToken` UUID 时序追溯校验机制，在提取的每页开始及写盘前进行校验，确保用户中途点击清除或快速重新导入新文件时，残留的旧后台工作线程能即时安全退出，彻底消除了旧数据写入新文档的交叉污染隐患。
- **TextEditor 双向绑定只读防护 (数据截断损毁 Bug 修复)**：重构了 TextEditor 计算 Binding 的 set 闭包使其完全只读。彻底阻断了当字数达到 15,000 字触发性能截断保护时，用户手抖敲击或删除字符会导致截断后的 15,000 字作为 newValue 回传进而毁灭性覆盖并抹除内存中完整长文本数据的致命 Bug。

## [1.4.0] - 2026-06-11

### Added
- **水印后台计算与安全防护**：将 `analyzeWatermarks` 移动至 `DispatchQueue.global().async` 异步扫描，消除拖入 PDF 时的 UI 卡顿。引入 `isAnalyzingWatermarks` 状态与 UUID `currentLoadToken` 时序追踪机制，彻底规避了用户在分析中途快速切换、清空或强行提取所造成的 PDFKit 读写竞态与 `EXC_BAD_ACCESS` 崩溃风险。
- **追加写盘安全加固与现代 API 升级**：优化每页提取后的自动写盘逻辑，弃用废弃 API 并改用现代 Swift 5 `seekToEnd()` 与 `write(contentsOf:)`。增加了完整的异常捕获与日志记录，即使文件在 Finder 中被意外删除，也能自动零异常重建，提升程序健壮性。
- **SSE 流式数据传输乱码彻底解决**：重构了流式网络数据接收代理，引入了字节级 `streamBuffer` 数据缓冲机制，通过识别换行符切分完整行。100% 解决了 TCP 粘包/分包以及 UTF-8 多字节字符被网络分片切断导致的 LLM 优化输出丢字与乱码 Bug。
- **URLSession 强引用循环内存泄露修复**：在 AI 推理结束（不论成功或中断）的代理方法中，显式调用 `invalidateAndCancel()` 销毁 Session。彻底切断了 `URLSession` 对代理对象（Engine Class）的永久强引用循环，确保内存资源在文件清空时能被正常回收。
- **UI 交互防重入锁**：当水印后台扫描正在进行时，“开始提取文字”按钮会自动展现流光等待动效并进入 `.disabled` 物理防刷状态，杜绝由于多次快速点击引起的逻辑冲突。

## [1.3.0] - 2026-06-11

### Added
- **AI 净化文本自动保存**：AI 推理优化顺利完成时，系统会自动在 PDF 文件同目录下创建名为 `[原文件名]_AI净化.txt` 的文本，并写入全量纠错排版后的结果。
- **高亮双色 UI Banner 提醒**：
  - 在“原始提取文本”Tab 顶部增加精致的绿色横幅卡片，显眼提示提取成果成功自动导出。
  - 在“本地 AI 纠错”Tab 顶部增加精致的紫色横幅卡片，显眼提示 AI 优化成果成功自动导出。
- **Finder 文件快速定位**：
  - 顶部提示横幅中显示导出的文件名，用户直接鼠标点击文件名，即可**一键激活并在 Finder 中选中高亮定位该文件**，极大地改善了原提示不明显的交互体验。
  - 优化了 clear 清理逻辑，切换/清空文件时重置所有导出的文件 URL 缓存。
- **二次元大脸专属 App 图标**：集成了以 `app_icon.png` 原始资产，利用 `sips -s format png` 强制转码并以 `iconutil` 编译出符合 macOS 官方 Retina/非 Retina 精准规格的 `AppIcon.icns` 并自动打包并挂载进应用包 `Resources` 目录的构建逻辑。

## [1.2.0] - 2026-06-11

### Added
- **无限超时机制**：将 AI 净化服务请求的超时阈值放宽至 `86400.0` 秒 (24 小时)，从机制上等同于“无限等待”，彻底免除了本地推理过程耗时长所导致的 Request Timeout 超时报错。
- **任务取消控制**：实现了本地 AI 文本净化的主动取消交互（cancel）。
  - 在 `PDFExtractorEngine` 中引入了 `currentAITask` 任务句柄。
  - 在 UI 的 AI 推理等待卡片中增加了一个精致的红色“取消优化”按钮，用户可随时中止推理。
- **生硬断行修复**：深度优化了本地 AI 排版净化提示词。在 System Prompt 中集成了“修复不合理的强行换行与分段”Few-shot 指示，指挥模型自动拼接合并由于 OCR 识字导致的碎行，使其重构为连贯流畅的长句及自然段，并在旁边输出 `【识别是：[硬换行]，修改为：[合并文本]】` 格式的大括号对比。
- **获取模型超时放宽**：将拉取本地模型列表的超时由 4 秒放宽至 15 秒，以更好支持休眠状态下本地 AI 服务的唤起响应。

## [1.1.0] - 2026-06-11

### Added
- **本地 AI 净化助手**：集成了兼容 OpenAI 规范的本地 AI 文本校对优化功能（如 LM Studio 和 Ollama）。
  - 支持 `GET /models` 获取本地服务已下载的可用大模型列表，并能够快速选择切换。
  - 提供了自动与手动填入模型名称的双重防错输入设计。
  - 内置了高精度 Few-shot 排版纠错提示词，告诉模型只需修正 OCR 笔误且不改变原文结构，并且强制使用 `【识别是：X，修改为：Y】` 大括号展示字词修改对比。
- **UI 双栏与 Tab 对比**：右侧正文展示区重构为卡片 Tab 栏，支持在“原始提取文本”和“本地 AI 净化文本”之间无缝切换。
  - 新增了 AI 推理状态展示与思考日志。
  - 提供了 AI 净化文本的一键复制功能。
- **HTTP 安全连接允许**：在 `Info.plist` 中追加 `NSAppTransportSecurity -> NSAllowsArbitraryLoads`，以确保能在沙盒或系统限制下与本地本地环回 HTTP API (Ollama/LM Studio) 进行畅通连接。

### Security
- 本地 AI 请求完全基于 localhost 进行，不经过任何外部公网代理，继续保障 PDF 文档的绝对私密。

## [1.0.0] - 2026-06-11

### Added
- **核心提取引擎**：实现了基于 `PDFKit` 与 `Vision` 框架的本地 PDF 文字提取。
  - 支持自动分析高频疑似水印。
  - 支持智能通道选择（活字直接提取、扫描件自动调起本地 OCR 识别）。
  - 支持无损文本净化（文本级精准剔除水印）和细粒度图片擦除（字符级像素填充白色遮蔽）双重去水印保障，解决遮盖正文图片的问题。
- **UI 界面**：基于 `SwiftUI` 实现了磨砂质感的 macOS 双栏交互界面。
  - 提供了 PDF 拖拽/点击文件导入框，智能卡片展示文件元数据。
  - 提供了疑似水印多选列表及自定义过滤词输入文本域。
  - 提供了提取状态运行日志实时输出 and 流光环形进度条。
  - 提供了提取文本大字号预览编辑器、一键复制到剪贴板，以及在 Finder 中定位导出 TXT 文件的功能。
- **编译与分发**：
  - 编写了 `build.sh` 一键打包脚本，能够将 Swift 单文件编译并自动打包为符合 macOS bundle 规范的 `PDF文字提取.app`。
  - 编写了应用元数据描述文件 `Info.plist`。

### Security
- 全功能 100% 运行于本地沙盒中，完全离线化无需网络权限，确保处理 of PDF 文档数据绝对保密、不上传云端。
