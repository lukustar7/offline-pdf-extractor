# 变更日志 (Changelog)

本项目的所有重大变更都将记录在此文件中。

本项目遵循 [语义化版本控制 (SemVer)](https://semver.org/spec/v2.0.0.html) 规范。

## [1.7.0] - 2026-06-15

### Added
- **水印模式三级单选机制**：在左侧设置面板新增了去水印工作模式选择组件，提供“智能诊断匹配（自动）”、“模式 A：纯文本过滤（文字版 PDF 专用）”、“模式 B：物理遮罩 + OCR（正文扫描件 + 文字水印）”、“模式 C：OCR + 智能过滤（纯扫描件水印）”的选项，并附带简要功能引导与模式 C 性能提示。
- **水印过滤全局控制开关**：在左侧设置面板新增一键启用/禁用去水印匹配机制的开关。用户对于完全无水印的文档可一键关闭，彻底消除误杀和 CPU 空转性能开销。
- **Keychain 加密凭证安全存储**：新增 `KeychainHelper` 系统安全钥匙箱管理，用户配置的 AI API Key 强制使用 Keychain 密文保护，彻底清除了原 `UserDefaults` 暴露明文 plist 文件的安全风险。
- **局域网网络传输豁免 (ATS 升级)**：在 `Info.plist` 中启用 `NSAllowsLocalNetworking`，完美豁免内网 IP 段（如 `192.168.x.x`）的 HTTP Ollama/LM Studio 流量，解决苹果安全机制对内网连接的静默阻断。

### Fixed
- **多线程并发 Data Race 崩溃修复 (Critical)**：将 AI 流式输出接收缓冲、界面更新刷新等状态逻辑完全收归在主线程（`DispatchQueue.main.async`）或串行处理，解决了大长文本在 Thread Sanitizer 下因高频并发竞争造成的随机闪退 Bug。
- **主线程磁盘 I/O 阻塞假死修复 (Critical)**：新建专职的后台 `ioQueue` 处理分片文件落盘与全量保存，将文字提取时空文件的初始化写入彻底移入 `pdfQueue` 后台线程。完全消除了在慢速介质或超大文本写入时，UI 主线程被同步磁盘 I/O 拖累而转彩虹球假死的顽疾。
- **SwiftUI 列表越界崩溃修复 (Critical)**：将水印配置的 ForEach 索引绑定迭代升级为基于 `Identifiable` 实体元素的直接迭代。彻底解决了由于 PDF 文档切换、候选水印数组瞬时缩小导致 SwiftUI 渲染越界 Crash 的高危隐患。
- **快捷键 ⌘R 冲突防重入拦截**：在快捷键提取逻辑中增加了对 `aiEngine.isAIProcessing` 等状态的比对，阻止了用户在 AI 推理中途通过快捷键重入文字提取引起的文件句柄竞争与数据覆盖。
- **页码范围全角连接符兼容**：升级 `parsePageRange` 页码解析算法，支持中英文全角半角连接符 `[-~～—]` 的自适应匹配，避免用户输入范围符号时因解析失败导致范围设置失效。

## [1.6.0] - 2026-06-15

### Added
- **PDF 页面与提取文本左右并排对照布局 (P1 提取前后对比核心实现)**：重构了右侧主展示区，使用 `HSplitView` 实现了 **左右并排双栏展示**。左半边常驻显示 PDF 原件预览，右半边显示文本（可在“原始提取文本”和“AI 净化文本”间切换）。用户可以直接在同一屏上并排对照 PDF 原图与提取出的正文，极大地提升了提取质量校验的易用性。
- **PDF 原生预览面板 (P0)**：右侧新增基于 PDFKit 的原生 PDF 渲染面板，支持双指缩放和滚动。
- **提取页码范围选择 (P1)**：在提取设置中新增页码范围输入框（支持 `1-5, 8, 10-12` 等语法格式），仅提取指定页码，为长文档处理节省大量等待时间。
- **本地常用 OCR 错别字纠错词库 (P2)**：内置高频 OCR 识别错字本地字典，在非 AI 模式提取完成后自动做匹配纠正。
- **预计剩余时间 (ETA) 动态显示 (P2)**：在进度环下方新增 ETA 预计剩余时间，根据单页处理均速动态更新。
- **键盘快捷键支持 (P2)**：支持窗口级快捷键：`⌘O` 快速导入文件，`⌘R` 快速触发文字提取。
- **可拖拽侧边栏宽度设计**：引入 `HSplitView` 弹性布局，支持用户拖拽调整左侧控制面板 of 的宽度（280pt - 500pt 限制）。
- **复制操作“已复制 ✓”反馈动画**：为“复制原始”和“复制净化文本”按钮增加短暂的绿色“已复制 ✓”状态提醒，2 秒后自动恢复。
- **用户参数及 AI 设置持久化 (@AppStorage)**：对提取模式、大小写、擦除水印、页码范围、自定义过滤词、AI 端点、默认模型、提示词等参数做持久化保存。
- **API 基础 URL 安全白名单校验与警示 (4.3)**：在 AI 设置模块中增加了本地地址白名单检测。若配置了外部公网 API 地址（非 localhost 或 192.168/10/172 内网网段），会在 API 输入框下方即时弹出红色的安全警告，提醒数据上传公网的潜在泄露风险。
- **水印扫描 counts 字典容量限制 (3.2)**：为后台高频词统计 counts 字典设定了 5000 条键的上限，超过限制则只做自增不插入新词。从算法上保障了水印分析的内存复杂度为恒定的 \(O(1)\)，避免在大样本下导致内存泄泄露。
- **导出格式单一问题彻底修复 (5.6)**：重构提取与 AI 推理落盘逻辑。现在提取正文时会同时自动生成同路径下的 `.txt` 和格式更精美的 `.md` (Markdown) 两份文件；AI 优化完成时也会同时自动生成 `_AI净化.txt` 和 `_AI净化.md` 两份文件。同时在顶部提示 Banner 中并排提供了这两个文件的 Finder 直达定位链接。

### Changed
- **单体巨石文件解耦重构**：将 2010 行的 `main.swift` 单体代码重构拆分为 `Sources/` 目录下的 11 个高内聚、扁平化的 Swift 文件，对应调整 `build.sh` 编译配置。
- **网络流式代理 delegate 架构解耦 (1.3)**：将流式数据解析 delegate 抽离并封装为独立的 `AIStreamDelegate: NSObject, URLSessionDataDelegate`，`AIProcessingEngine` 变回了遵循 `ObservableObject` 的纯 Swift 视图模型类，消除了 `NSObject` + `ObservableObject` 混合继承的设计缺陷。
- **深色模式恢复为自适应系统主题**：移除了强制的 `.dark` 深色模式渲染，完美适配系统浅色与深色主题。
- **窗口标准标题栏恢复**：移过了隐藏标题栏，使红黄绿交通灯正常显现，窗口拖拽与常规一致。

### Fixed
- **多线程 PDFKit 访问崩溃隐患 (Critical)**：后台提取与 OCR 渲染采用单独加载的 `PDFDocument(url: url)` 实例，与主线程 UI 实现物理隔离，彻底规避了多线程并发读写 PDFKit 导致崩溃的 Bug。
- **@Published 属性跨线程写入 Bug (Critical)**：对 cancellation 状态添加了 `NSLock` 保护并提供线程安全的 `isCancelledSafe` 接口；所有对 `@Published` 修改与 UI 更新均显式分发到主线程异步执行。
- **Ollama/LM Studio 流式 UI 更新节流 (Throttle)**：实现 100ms 字符接收缓冲区，将 SSE 流的界面刷新频次压缩了 90% 以上，解决了 TextEditor 频繁重绘带来的严重 UI 卡顿问题。
- **Vision OCR 信号量死锁隐患**：将 OCR 异步回调阻塞改为 `timeout: .now() + 60.0` 限时等待，若超时则输出错误日志并自动处理下一页，防止程序因系统 Vision 故障永久锁死在后台。
- **FileInfoView SF Symbol 图标空白修复**：将不存在的 `"doc.red"` 更正为有效的 `"doc.fill"`。
- **“提取模式”单选组标签隐藏 Bug 修复**：通过外置显式渲染在单选组上方，解决 `labelsHidden` 将其含义完全抹除的问题。

## [1.5.1] - 2026-06-11

### Fixed
- **自然段落与换行破坏修复 (排版丢失修复)**：重构了本地 AI 推理的 `systemPrompt` 与 `strictSystemPrompt`，将“保留段落结构与原有换行”升级为【核心铁律】，限制大模型激进的硬换行合并逻辑。在分片拼接以及追加落盘时，将段与段之间的连接符升级为双换行 `\n\n` 进行物理隔离，彻底解决了长文档优化后被强行压缩成一整大段的排版 Bug。
- **无遮挡流式打字机 UI 未编译生效修复 (编译发版)**：排查发现上轮虽然重构了 UI 的 `TextEditor` 自动切菊花展现逻辑，但未执行 `./build.sh` 重新编译生成 macOS 应用包，导致用户运行的仍是旧版 App。本次完成修改后已全盘重新编译生成最新的 `PDF文字提取.app` 并进行本地验证。
- **SSE 流式数据多态解析容错**：优化了代理层接收流数据的 JSON 提取逻辑，新增对非标准 SSE 流（如部分本地 AI 引擎在 stream 模式下返回的无 `data: ` 前缀纯 JSON 块）的自适应解析兼容，并提供了详尽的解析异常与 DONE 标志控制日志，全面加固流式出字在各种离线模型下的稳定性。

## [1.5.0] - 2026-06-11

### Added
- **智能长文本分片推理 (Context OOM & Truncation 修复)**：实现智能多级退避分片算法，在 1500~2000 字窗口内优先按换行与常见标点符号断句断段，预留硬物理切片保命红线。彻底免除了超长 PDF 一次性发送给本地 AI 时造成的显卡爆显存崩溃或上下文超载被强行截断的重大隐患。
- **串行队列推理与分片追加落盘**：底层 AI 采用串行 FIFO 队列流式跑分片，恒定极低推理负载，低配置显卡亦能流畅应对大作。每个分片推理完立即以追加模式落盘 `_AI净化.txt` 并提供被删重建保护，大幅提升容错性。
- **无遮挡打字机流式回显 UI**：重构了 AI tab 界面展现逻辑，仅在首字吐出前的“热身期”显示大转菊花；一旦收到首字，立刻撤走菊花展示文本编辑器，让文字在用户眼前实时流畅流下。
- **流光分段进度横幅**：在 AI 文本框上方新增紫色常驻横幅，动态提示 `“⚡️ AI 正在接力流式净化中：第 X / Y 段...”` 及 `[取消优化]` 按钮，赋予用户清晰的处理盼头与交互主导权。
- **OCR 大图渲染内存强制释放 (提取 OOM 修复)**：在提取循环体的页面处理外围增加 `autoreleasepool` 物理强放块，确保每页产生的高清 NSImage 和 NSBitmapImageRep 在用完后即刻被销毁，使超长 PDF 提取内存保持恒定平稳。
- **多线程写盘防交叉污染安全防护 (残留时序 Bug 修复)**：设计 `currentExtractToken` UUID 时序追溯校验机制，在提取的每页开始及写盘前进行校验，确保用户中途点击清除或快速重新导入新文件时，残留的旧后台工作线程能即时安全退出，彻底消除了旧数据写入新文档的交叉污染隐患。
- **TextEditor 双向绑定只读防护 (数据截断损毁 Bug 修复)**：重构了 TextEditor 计算 Binding 的 set 闭包使其完全只读。彻底阻断了当字数达到 15,000 字触发性能截断保护时，用户手抖敲击或删除字符会导致截断后的 15,000 字作为 newValue 回传进而毁灭性覆盖并抹除内存中完整长文本数据的致命 Bug。

## [1.4.0] - 2026-06-11

### Added
- **水印后台计算与安全防护**：将 `analyzeWatermarks` 移动至 `DispatchQueue.global().async` 异步扫描，消除拖入 PDF 时的 UI 卡顿。引入 `isAnalyzingWatermarks` 状态与 UUID `currentLoadToken` 时序追踪机制，彻底规避了用户在分析中途快速切换、清空或强行提取所造成的 PDFKit 读写竞态与 `EXC_BAD_ACCESS` 崩溃风险。
- **追加写盘安全加固与现代 API 升级**：优化每页提取后的自动写盘逻辑，弃用废弃 API 并改用现代 Swift 5 `seekToEnd()` 与 `write(contentsOf:)`。增加了完整的异常捕获与日志记录，即使文件在 Finder 中被意外删除，也能自动零异常重建，提升程序健壮性。
- **SSE 流式数据传输乱码彻底解决**：重构了流式网络数据接收代理，引入了字节级 `streamBuffer` 数据缓冲机制，通过识别换行符切分完整行。100% 解决了 TCP 粘包/分包以及 UTF-8 多字节字符被网络分片切断导致的 LLM 优化输出丢字与乱码 Bug。
- **URLSession 强引用循环内存泄露修复**：在 AI 推理结束（不论成功或中断）的代理方法中，显式调用 `invalidateAndCancel()` 销毁 Session。彻底切断了 `URLSession` 对代理对象（Engine Class）的永久强引用循环，确保内存资源在文件清空时能被正常回收。
- **UI 交互防重入锁**：当水印后台扫描正在进行时，“开始提取文字”按钮会自动展现流光等待动效并进入 `.disabled` 物理防刷状态，杜绝由于多次快速点击引起的逻辑冲突。

- **获取模型超时放宽**：将拉取本地模型列表的超时由 4 秒放宽至 15 秒，以更好支持休眠状态下本地 AI 服务的唤起响应。

## [1.0.0] - 2026-06-11

### Added
- **核心提取引擎**：实现了基于 `PDFKit` 与 `Vision` 框架的本地 PDF 文字提取。
-   支持自动分析高频疑似水印。
-   支持智能通道选择（活字直接提取、扫描件自动调起本地 OCR 识别）。
-   支持无损文本净化（文本级精准剔除水印）和细粒度图片擦除（字符级像素填充白色遮蔽）双重去水印保障，解决遮盖正文图片的问题。
- **UI 界面**：基于 `SwiftUI` 实现了磨砂质感的 macOS 双栏交互界面。
-   提供了 PDF 拖拽/点击文件导入框，智能卡片展示文件元数据。
-   提供了疑似水印多选列表及自定义过滤词输入文本域。
-   提供了提取状态运行日志实时输出 and 流光环形进度条。
-   提供了提取文本大字号预览编辑器、一键复制到剪贴板，以及在 Finder 中定位导出 TXT 文件的功能。

-   编写了 `build.sh` 一键打包脚本，能够将 Swift 单文件编译并自动打包为符合 macOS bundle 规范的 `PDF文字提取.app`。
-   编写了应用元数据描述文件 `Info.plist`。

### Security
- 全功能 100% 运行于本地沙盒中，完全离线化无需网络权限，确保处理 of PDF 文档数据绝对保密、不上传云端。
